Testing Type
Purpose
What Will Be Tested
Method


Unit Testing
Ensure individual components work correctly before integration
Login system, assignment submission module, messaging system, progress tracking module
Developers test each module independently using test data to verify expected outputs


Integration Testing
Ensure system components work correctly when combined
Login connecting to user database, assignments linking to student accounts, messaging storing chat history
Modules are connected together and tested to confirm correct communication between components


System Testing
Verify the complete LMS works correctly as a full platform
Course access, assignment uploads, communication tools, dashboards, calendars
Full system testing in a simulated environment with multiple user accounts


Security Testing
Protect student and tutor data and prevent unauthorised access
Secure login, role-based access control, data encryption, database protection
Attempted unauthorised access tests and verification of encryption and authentication systems


Usability Testing
Ensure the system is easy for tutors and students to use
Navigation, assignment submission process, resource access, messaging
Real users test the platform and provide feedback on usability


Performance Testing
Ensure the system performs well under load
Multiple users accessing resources, submitting assignments, and messaging simultaneously
Simulate multiple users accessing the system and monitor response time and system stability


Accessibility Testing
Ensure the system meets WCAG accessibility standards
Screen reader compatibility, keyboard navigation, colour contrast, text resizing
Test system using accessibility tools and assistive technologies
